
Для запуска сервера нужно перейти в директорию R4C в корень запустить файл start из командной строки -> python start.py
в файли start.py можно запустить как отладочную версию -> 1, так и рабочую на uvicorn -> 2 (раскомментируйте нужный вариант)

По  заданиям :

1) task 1 http://localhost:8000/add_robot/ методом POST  кидаем json фомата {"model"str, "version":str "created":str}, при успешном запросе получаем ответ -> {'message': 'Робот успешно добавден в БД'}

2) task 2 http://localhost:8000/report/ прямая ссылка, через браузер вводим адрес получаем отчет. Для формирование отчета используется библиотека pandas.

3) task 3 http://localhost:8000/order_a_robot/ методом POST  кидаем json фомата {"email":str, "robot_serial":str}, 
при успешном запросе получаем ответ -> Либо {'message': 'Робот по вашему заказу будет скоро доставлен'}) если есть сводобный робот, {'message': 'Робот по вашему заказу будет скоро доставлен'})
Либо {'message': 'Дорогой заказчик настоящий момент у на нет данного работа в наличие(, как только он появится мы сообщим вам по почте'}, если свободного робота нет

По механизму task 3 если мы оставили успешный запрос и робот есть в наличие робот перейдем в цех упаковки для этого в модели Robot добален status 
если подходящего робота нет или есть робот но он уже в упаковке для для клиента мы заказ создаем как ожидающий так же в модель Order добален status для этой функции
как только будет создан новый робот клиенты из ожидающих заказов получат уведомляния по почте статус заказов по созданному роботум изменится на  на отправллено письмо
и мы предпологаем что если для клиента предложение еще актуально он заного оформит заказ.






Для настройки почты нужно прописать настройки почты в .R4C/settings.py в конце файла. Проверял на mail.ru
